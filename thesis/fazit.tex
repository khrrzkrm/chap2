\chapter{Conclusion and Future Work}
\label{chapter:conclusion}

\section{Summary}
This dissertation explores key challenges in using formal methods to digitalize and automate the analysis of normative systems.
As more laws, regulations, and contracts guide how autonomous software agents act, the vagueness of natural language becomes a challenge.
To address this challenge, we demonstrate that Normative System Engineering can achieve practical efficacy by maintaining a controlled expressiveness in the underlying logic.
By restricting complexity to ensure tractable reasoning, we can automate critical tasks such as conflict detection and blame attribution. Without this principled restriction, these tasks remain computationally intractable in general settings.

The first main contribution, covered in Chapter 3, concerns the static analysis of normative specifications. We introduced Micro Metric-Time Normative Logic (\TDDLfm) to formalize the identification of conflicts between time-constrained obligations and prohibitions. We developed a special normal form, called Disjunctive Punctual Normal Form (\dpnf), to account for timing conditions by breaking down complex time rules into simple, checkable parts while preserving key properties such as the Lowest Upper Common Operator (\LUCO) and Literal Ancestor Preservation (\LAP). This setup lets us analyze inconsistencies in detail, revealing a direct link between the conflicts found and Minimal Unsatisfiable Subsets (MUS). By tying normative requirements to these basic contradictions, we could clearly separate deontic conflicts, i.e., normative contradictions, from ontic conflicts, i.e., physical impossibilities. We also defined a measure of conflict density and developed a method to remove it. These tools help regulators check and fix rules before they are used, making sure they are clear and practical.

The second main contribution, discussed in Chapter 4, is about monitoring collaborative interactions during contract enforcement. We developed Two-Agents Collaborative Normative Logic (\cDL) to model contracts where following the rules depends on the joint actions of several parties. A key part of this framework is that it clearly separates attempts from collaboration outcomes. This distinction allows the monitors to correctly attribute blame to the right agents.

To fully explain how interactions between violations and blames are propagated, we built a layered semantic framework. First, we made a forward-looking method to detect exactly when satisfaction or violation happens, using deterministic Moore machines. We also went beyond simple yes-or-no monitoring verdicts by adding Quantitative Blame Semantics, powered by a Contract Progress Monitor. This monitoring approach tracks how contracts change after the first breach, prompting monitors to count the blame score afterward. By turning these costs into blame vectors for each agent, the framework gives a detailed view of responsibility, allowing it to explain each failure. Putting these ideas into executable Mealy machines provides a reliable and practical way to track normative rules in real time and settle disputes fairly afterward.

Together, these chapters provide an initial formal methodology for automating parts of normative systems.

\section{Future Work}

This dissertation explored new research directions in the theory of formal semantics for normative reasoning. Future work will combine the main methods of conflict-aware temporal reasoning and responsibility-sensitive collaboration into more expressive, easier-to-use frameworks. Key objectives include expanding these foundations to handle more complex timing, multi-agent relationships, and ongoing compliance. These improvements will help link normative logic with real-world monitoring, supporting full verification, automated monitoring, and accountability in normative systems.

\subsection{Toward a Unified Formalism (\logic{MTTACNL})}

The next step for this research is to combine the two main formalisms developed in this dissertation.
Chapter 3 introduced \TDDLfm, which focuses on metric-time aspects of normative specifications in a single-agent setting, whereas Chapter 4 presented \cDL, which focuses on a two-agent setting with more operators and uses logical time. However, keeping them separate limits their use in situations that have both strict timing rules and cooperative dependencies.
To bridge this gap, a primary objective for future work is to develop a unified framework, tentatively styled as the \emph{Metric-Timed Two-Agents Collaborative Normative Logic} (\logic{MTTACNL}).

Creating this unified framework will require a complete rebuild of the underlying semantic models.
Specifically, the discrete-event traces used in our collaborative logic must be replaced by \emph{metric-timed traces} in which every collaborative action and every attempt are associated with a precise real-valued timestamp.
On the syntax side, the logic should smoothly combine interval-based constraints with collaborative operators.
This richer syntax allows specifying complex requirements, such as "Agent 1 must accomplish action $\acta$ within 5 time units without interference from Agent 2."

Putting this unified logic into practice will require a two-part approach to automated reasoning.

First, for static analysis, the conflict detection algorithms from Chapter 3 need to be adapted to handle interference between multiple agents.
This change would allow regulators to verify both that temporal windows remain feasible and that the contract does not force any agent into a position where they must actively interfere with another to fulfill their own duties.
Second, for dynamic enforcement, the monitoring system needs to move from standard Moore machines to timed automata or other real-time computational models.
Achieving this would provide a transparent and explanatory solution that simultaneously monitors adherence to deadlines and assigns blame for non-compliance accordingly.

\subsection{Beyond Discrete Actions}

In this dissertation, we worked with the assumption that actions are atomic and instantaneous.
While discrete actions do occur in practice and many real-world rules fit this model, many norms require actions that take place over a longer period, not just in a single moment.
A logical next step is to include actions that last for a set period, in which following the rules depends on maintaining the behavior over that period, as suggested by reviewers of the conference version of a published work.

Pushing this complexity further requires moving to the domain of \emph{continuous real-time actions}, a challenging counterpart to discrete logic, in which the system must monitor varying physical signals such as velocity or temperature rather than symbolic transitions.
To handle this continuous domain, one needs to develop clear abstractions that connect normative reasoning with control theory.

Looking beyond the timing aspect of actions brings us to the key difference between operational steps and declarative goals.
Our current formalisms are strictly ought-to-do logics, focusing only on the agents' transitions and interactions.
However, many real-world regulations are naturally expressed as \emph{ought-to-be} specifications that constrain the resulting system state rather than the specific means of achieving it.
For example, a safety regulation may mandate that a room remain below a certain temperature without dictating the specific actions of the cooling system.
Capturing the full spectrum of real-life normative specifications therefore requires a hybrid logic capable of reasoning about both the actions agents perform and the environmental states those actions induce.
